3).What is difference between Strict, Transitional and Frameset?

ANS. Strict:- which describes the structural portions of HTML and does not provide any presentational markup or frame-related markup. Transitional:- which includes the presentational markup such as and , but does not include frame-related markup. Transitional DTD allows some older PUBLIC and attributes that have been deprecated. Frameset:- which is the same as Transitional except that it adds a description of frame-related markup. If frames are used, the Frameset DTD must be used.

7). What are block-level and inline elemnts?

ANS. Block-level elements are usually 'containers' for other elements. Examples of block-level elements are DIV, P, FORM and TABLE. Some block-level elements, e.g., P, can only contain text and inline elements. Others, e.g., FORM, can only contain block-level elements (in the Strict DTD). And some, like DIV, can contain text, inline elements and block-level elements Inline elements are elements that can occur 'inline' within text. Examples: A, EM, Q and SPAN. An inline element can only contain text and other inline elements. An inline element cannot contain a block-level element, with one exception: OBJECT

9).What is the use of the ADDRESS element type?

ANS. To mark up contact information for the page (or for a part of a page). This can be a postal address, an email address, a telephone number, or virtually anything. ADDRESS is a block-level element which can only contain text and inline elements.

10). What is the use of the VAR element type?

ANS.To mark up a variable, or placeholder, part of an example. It is a typographic convention to italicise such variables, which will be replaced by actual data in real life use.
